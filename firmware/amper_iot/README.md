# Amper IoT — دليل التثبيت الكامل

> فيرموير ESP32 لمراقبة المولدات الكهربائية في نظام Amper.
> الجهاز يقرأ: **الحرارة + الوقود + التيار** ويرسلها للسيرفر كل دقيقة.

## 🔌 مخطط التوصيل

```
              ┌────────────────────┐
              │     ESP32 DevKit   │
              │                    │
   DS18B20 ──►│ GPIO 4   (TEMP)   │
              │ + 4.7kΩ to 3.3V   │
              │                    │
   HC-SR04   ─►│ GPIO 5   (TRIG)   │
              │ GPIO 18  (ECHO)   │
              │                    │
   SCT-013   ─►│ GPIO 34  (ADC)    │
              │ + burden resistor │
              │                    │
              │ 3.3V ──► sensors   │
              │ GND  ──► sensors   │
              │                    │
              │ USB ──► 5V power  │
              └────────────────────┘
```

### تفاصيل التوصيلات:

#### 1. حساس الحرارة DS18B20 (3 أسلاك)
```
DS18B20            ESP32
───────            ─────
أحمر (VCC)    ──►  3.3V
أسود (GND)    ──►  GND
أصفر (DATA)   ──►  GPIO 4
                   + مقاومة 4.7kΩ بين DATA و 3.3V
```

#### 2. حساس الوقود HC-SR04 (4 أسلاك)
```
HC-SR04            ESP32
───────            ─────
VCC           ──►  5V (Vin)
GND           ──►  GND
TRIG          ──►  GPIO 5
ECHO          ──►  GPIO 18
```

> ⚠️ **تحذير:** ECHO يطلع 5V — استخدم voltage divider بسيط (مقاومتان 1kΩ + 2kΩ) لحماية ESP32.

#### 3. حساس التيار SCT-013-030
```
SCT-013            ESP32
───────            ─────
سلك أبيض      ──►  GPIO 34 (ADC)
سلك أحمر      ──►  3.3V (عبر burden resistor 33Ω)
                   + مكثف 10µF للاستقرار
```

> 💡 **معايرة:** ضبطنا الكود على نطاق 0-30A. لو CT clamp مالك مختلف، عدّل `1V = 30A` في الكود.

#### 4. حساس الفولتية ZMPT101B
```
ZMPT101B           ESP32
────────           ─────
VCC           ──►  5V (Vin)
GND           ──►  GND
OUT           ──►  GPIO 35 (ADC)

⚡ توصيل الفولت العالي:
الطرفان الكبيران للحساس يوصلان مع مخرج المولدة (220V AC)
— مباشرة بالتوازي مع الحمل
```

> ⚠️ **خطر:** هذا حساس فولت عالي (220V AC). لا تلمسه أثناء التشغيل!
> اعزله جيداً داخل العلبة البلاستيكية.

> 💡 **معايرة:** عدّل قيمة `CALIBRATION` في الكود (الافتراضي 0.27)
> قارن قراءة الجهاز مع جهاز فولتميتر مرجعي وعدّل النسبة.

#### 5. التغذية
```
محول USB 5V / 2A  ──►  منفذ Micro-USB / USB-C على ESP32
```

## 💻 تثبيت بيئة التطوير (مرة واحدة فقط)

### 1. تنزيل Arduino IDE
- اذهب إلى https://www.arduino.cc/en/software
- نزّل النسخة المناسبة لنظامك (Windows/Mac/Linux)
- ثبّتها

### 2. إضافة دعم ESP32
1. افتح Arduino IDE
2. **File → Preferences**
3. في حقل **Additional Boards Manager URLs** الصق:
   ```
   https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
   ```
4. **OK**
5. **Tools → Board → Boards Manager**
6. ابحث عن **"esp32"** ثم **Install**

### 3. تثبيت المكتبات المطلوبة
**Tools → Manage Libraries** ثم ابحث وثبّت:

| المكتبة | الكاتب |
|---|---|
| `WiFiManager` | tzapu |
| `OneWire` | Paul Stoffregen |
| `DallasTemperature` | Miles Burton |
| `ArduinoJson` | Benoit Blanchon (v6+) |

## ⬆️ رفع الفيرموير على ESP32

### 1. اختر اللوحة
- **Tools → Board → ESP32 Arduino → ESP32 Dev Module**

### 2. إعدادات مهمة
| الإعداد | القيمة |
|---|---|
| Upload Speed | 115200 |
| Flash Frequency | 80MHz |
| Flash Size | 4MB (32Mb) |
| Partition Scheme | Default 4MB with spiffs |
| Port | اختر COM المناسب |

### 3. افتح الملف
- **File → Open** → اختر `firmware/amper_iot/amper_iot.ino`

### 4. اضغط Upload (السهم ←)
- انتظر حتى تشاهد: `Hash of data verified.`

> ✅ **الجهاز جاهز** — لا تحتاج تعديل أي شيء بالكود!

## 🚀 الإعداد الأول للجهاز (بدون كومبيوتر!)

بعد رفع الفيرموير مرة واحدة، الجهاز يعمل **بدون أي تعديل بالكود** للمستخدم النهائي:

### الخطوة 1 — إنشاء الجهاز في تطبيق Amper
1. افتح تطبيق Amper Manager
2. **الإعدادات → أجهزة IoT → ➕ إضافة جهاز**
3. اكتب اسم الجهاز (مثال: "حساس مولدة الكرادة")
4. اختر المولدة والمحركات المرتبطة
5. اضغط **"إنشاء + عرض QR"**
6. **سيظهر رمز من 6 خانات** (مثلاً: `483921`) — احفظه أو لا تغلق الشاشة

### الخطوة 2 — توصيل الكهرباء للجهاز
- وصّل ESP32 بالكهرباء (USB أو محول)
- LED يومض 3 مرات → الجهاز دخل **وضع الإعداد**

### الخطوة 3 — الاتصال بالجهاز من موبايلك
1. على موبايلك، افتح **إعدادات WiFi**
2. اختر شبكة اسمها `Amper-Setup-XXXXXX`
3. تنفتح صفحة تلقائية (إذا ما انفتحت، افتح المتصفح واذهب لـ `192.168.4.1`)

### الخطوة 4 — إعداد WiFi والإقران
في الصفحة المفتوحة:
1. اضغط **"Configure WiFi"**
2. اختر شبكة WiFi الموقع
3. اكتب الباسوورد
4. في حقل **"رمز الإقران Amper"** اكتب الرمز من الخطوة 1 (`483921`)
5. اضغط **Save**

### الخطوة 5 — انتهى!
- LED يومض 5 مرات → الإقران نجح ✓
- الجهاز يبدأ بإرسال البيانات كل دقيقة
- ارجع لتطبيق Amper → الداشبورد → **كرت "أجهزة IoT المباشرة"** → ستشاهد قراءاتك تظهر

## 🔄 إعادة الضبط (Factory Reset)

إذا أردت إعادة الجهاز للوضع الأولي:

1. اضغط واستمر بالضغط على زر **BOOT** على ESP32
2. استمر بالضغط لمدة **5 ثوانٍ**
3. LED يومض 10 مرات → الجهاز انمسح وأعاد التشغيل
4. ستحتاج إعادة الإقران (الخطوات 1-5 أعلاه)

## 🛠️ استكشاف الأعطال

### المشكلة: شبكة `Amper-Setup-XXXX` لا تظهر
**الحل:**
- تأكد ESP32 موصول بالطاقة (LED يومض)
- اضغط زر RESET (EN) على ESP32
- إذا كان الجهاز مقترن مسبقاً، استخدم Factory Reset (5 ثوانٍ على BOOT)

### المشكلة: الإقران فشل
**الحل:**
- تأكد من رمز الـ 6 خانات صحيح (انتبه — حساس للحالة)
- الرمز صالح **15 دقيقة فقط** — إذا انتهى، ولّد رمز جديد من التطبيق
- تأكد WiFi الذي اخترته يصل للإنترنت

### المشكلة: الجهاز لا يرسل بيانات
**الحل:**
- افتح **Serial Monitor** في Arduino IDE (115200 baud)
- شاهد الـ logs — ستظهر رسائل HTTP و قراءات الحساسات
- تأكد ESP32 متصل بـ WiFi (`WL_CONNECTED`)
- تأكد رابط السيرفر صحيح في `SERVER_URL`

### المشكلة: قراءة الحرارة غريبة (`-127°` أو `nan`)
**الحل:**
- تأكد من المقاومة 4.7kΩ بين DATA و 3.3V
- جرّب pin آخر (مثلاً GPIO 16)
- تحقق من توصيلات DS18B20 (VCC و GND)

### المشكلة: قراءة الوقود غير دقيقة
**الحل:**
- HC-SR04 يعمل بـ 5V — تأكد من توصيله بـ Vin مو 3.3V
- استخدم voltage divider على ECHO لحماية ESP32
- عدّل `TANK_EMPTY_CM` و `TANK_FULL_CM` في الكود حسب خزانك

## 📊 ما الذي يرسله الجهاز؟

كل **60 ثانية**، الجهاز يرسل:
```json
{
  "engine_id": "<uuid>",
  "temperature_c": 73.5,
  "fuel_pct": 68,
  "current_a": 18.2,
  "run_status": true
}
```

كل **5 دقائق**، يرسل heartbeat (إشارة "أنا حي"):
```json
{ "firmware": "1.0.0" }
```

إذا الجهاز ما أرسل heartbeat لمدة **5 دقائق**، النظام يعتبره **"غير متصل"** ويرسل تنبيه للمدير.
